# Polis Governance

## Overview

Polis is governed as an open-source project under the MIT licence, with
the long-term goal of transitioning to a community-managed foundation.

## Decision Making

| Scope | Process |
|-------|---------|
| Bug fixes & minor improvements | Pull Request review by one maintainer |
| New features & API changes | RFC in `specs/` + two maintainer approvals |
| Invariant changes | RFC + unanimous maintainer approval + 14-day comment period |
| Protocol version bumps | RFC + community vote (when foundation is established) |

## Roles

- **Maintainers**: Core contributors with merge rights. Responsible for
  code review, release management, and enforcing invariants.
- **Contributors**: Anyone who submits a pull request or issue.
- **Moderators**: Protocol-level moderator identities that sign moderation
  records (see `specs/moderation/`).

## Code of Conduct

All participants are expected to follow the
[Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

## Foundation Roadmap

Phase 4 of the Polis roadmap includes the establishment of a non-profit
foundation to hold protocol governance, manage the DID method registry,
and coordinate protocol upgrades. See [README.md](README.md) for the
full roadmap.

## Amendments

This governance document can be amended via the RFC process described
above. All changes require a 14-day comment period.
